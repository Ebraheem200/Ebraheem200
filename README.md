## Featured Projects
### bchoc — Blockchain Chain of Custody System

`bchoc` is a command-line blockchain-style evidence tracking system built in Python. It stores evidence records in an append-only binary file and verifies the integrity of the chain using SHA-256 hashes, encrypted identifiers, and a strict item state machine.

The project simulates a digital chain-of-custody workflow where evidence can be added, checked out, checked in, removed, summarized, and verified.

**Tech Used**

![Python](https://img.shields.io/badge/Python-3776AB?style=for-the-badge&logo=python&logoColor=white)
![CLI](https://img.shields.io/badge/CLI-Command_Line-black?style=for-the-badge)
![Cryptography](https://img.shields.io/badge/Cryptography-AES%20%2B%20SHA--256-blue?style=for-the-badge)
![Binary I/O](https://img.shields.io/badge/Binary%20I%2FO-Append--Only-orange?style=for-the-badge)

**Key Features**

- Built a Python command-line interface using `argparse`
- Implemented an append-only binary storage format for blockchain records
- Used SHA-256 hashing to verify block links and file integrity
- Added AES-based encryption helpers for case IDs and item IDs
- Enforced evidence state transitions such as `CHECKEDIN`, `CHECKEDOUT`, `DISPOSED`, `DESTROYED`, and `RELEASED`
- Supported role-based password checks through environment variables
- Added commands for initializing, adding, checking in, checking out, removing, showing, summarizing, and verifying evidence records
- Implemented chain verification to detect broken hashes, invalid file structure, and illegal item state changes
